# Presentation Deck Notes

This repository is a working space for developing a talk about AI/LLM expectations, practical usage, risks, and long-term impact. It is organized from loose idea capture to structured outline to an iterative manuscript.

## What This Repo Is

This is a presentation-writing knowledge base, not a software project. The files are meant to support:

- idea collection from source content
- theme and section structuring
- iterative drafting with evidence-backed expansion
- eventual slide condensation and speaking flow prep

## Repository Map

- `new_outline.md`
  - Big-picture structure of the talk.
  - Defines major sections and rough talking points.
  - Best starting point for understanding scope and flow.

- `iterative_full_draft.md`
  - Expanded manuscript built from the outline.
  - Organized by section iterations with intent, explanation, and sourced quotes.
  - Includes carry-forward decisions and unresolved follow-ups for future refinement.

- `input_notes/`
  - Raw notes, scribbles, links, quotes, and rough observations.
  - Main source material feeding the manuscript.
  - Contains:
    - `input_notes/notes_1.md`
    - `input_notes/notes_2.md`
    - `input_notes/notes_3.md`

## Suggested Workflow

1. Start in `new_outline.md`
   - Confirm section order, narrative arc, and key claims.

2. Expand or revise in `iterative_full_draft.md`
   - Develop each outline line into clear audience-facing language.
   - Preserve practical framing and avoid drifting into speculative tangents unless intentional.

3. Ground claims in `input_notes/`
   - Trace important assertions back to notes and linked sources.
   - Keep evidence explicit when refining claims with higher risk of overstatement.

4. Condense for deck delivery
   - Convert manuscript sections into short slide points and speaker notes.
   - Favor clarity, calibration, and actionable takeaways over maximal detail.

## Writing Conventions

Current writing style in this workspace emphasizes:

- calibration over hype or dismissal
- practical software-oriented framing
- explicit treatment of limitations and risks
- fair representation of opposing views
- iterative refinement over one-pass perfection
